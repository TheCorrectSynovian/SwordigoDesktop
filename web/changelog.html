🎨 Advanced Rendering Pipeline

SSAO
Screen Space Ambient Occlusion — 16-sample hemisphere with bilateral blur at half resolution
God Rays
64-sample screen-space radial blur from configurable sun position with depth masking
Volumetric Light
Depth-masked crepuscular light shafts with warm golden tint
7 Visual Presets
Off, Cinematic, Retro, Fantasy, Noir, Ethereal, Atmospheric — cycle with F6
Color PostFX
Vignette, Film Grain, Chromatic Aberration, Color Grading, Sharpen
Depth Texture
Depth buffer converted from renderbuffer to texture for shader sampling

🚀 Boot & Binary System

Unified Launcher
Pre-launch GUI with binary selection and graphics API picker
Binary Selector
SHA-256 validated multi-binary support with auto-detection
JSON Registry
Persistent version tracking with tested/untested status per binary

⚡ Engine Improvements

  • 5-pass rendering pipeline: SSAO → God Rays → Composite → Color FX → Upscale
  • Half-resolution FBOs for SSAO and God Rays (performance optimization)
  • 8 GLSL shader programs compiled at init
  • GUI click detection fix (1px sidebar offset corrected)
  • glGetFloatv JNI bridge handler added
  • Dropdown menus now properly block game input (has_modal_open fix)
  • Vulkan backend wiring (ifdef'd, experimental)
  • Window title updated to "Swordigo Desktop v2.0r"

✅ What's Working

  • Main menu rendering
  • In-game level rendering
  • Gameplay and character movement
  • Keyboard and mouse controls (to some extent)
  • Sound effects
  • OpenGL desktop renderer
  • Asset loading and scene loading
  • Linux-native execution through custom compatibility runtime

⚙️ Technical Notes

  • Runs the original ARM Android game library through a custom Linux runtime
  • Implements Android JNI and asset system replacements
  • Translates OpenGL ES 1.x rendering to desktop OpenGL
  • Uses native Linux windowing, input, and audio backends
  • Tested primarily on modern Linux distributions

⚠️ Known Issues

  • Save data is currently not written correctly and may not persist between sessions
  • Background music playback is unreliable and may fail to start
  • Music issues are more common in the AppImage build
  • Minor graphical artifacts may occasionally appear
  • Some Android-specific features remain stubbed or partially implemented

📝 Release Notes

This is an early beta release intended for testing and feedback.


The game is playable, but some systems are still under active development. Future releases will focus on save support, audio stability, controller support, and general polish.


Thanks to everyone who tested and helped debug this project. ⚔️
