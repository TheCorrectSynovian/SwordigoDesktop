Architecture

How It Works

A custom compatibility stack bridges the original ARM Android game engine to your Linux desktop.

🎮
Your Linux Desktop
SDL2 window, OpenGL context, OpenAL audio, native input
↕
🔧
Compatibility Runtime
JNI bridge, Android API shims, asset manager, log stubs
↕
⚙️
ARM Emulation (Unicorn)
ARMv7 instruction execution, memory mapping, relocation
↕
📦
libswordigo.so (ARM)
Original game engine binary — Caver engine, Lua scripting, protobuf data

Technical Highlights

  • Runs original ARM Android game library — no recompilation
  • Custom ELF loader with full relocation support
  • 254 external symbols resolved via bridge trampolines
  • Fake JNI environment with 40+ implemented functions
  • OpenGL ES 1.x → Desktop OpenGL translation
  • Android AAssetManager → filesystem shim
  • OpenAL for positional audio & sound effects
  • Lua 5.1 scripting engine embedded in the game
  • Protobuf scene / game data parsing
  • Based on research from PS Vita port architecture
