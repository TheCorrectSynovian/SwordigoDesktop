⚙️ Installation Guide

The AppImage is self-contained. Just make it executable and run it:

# Download the .deb package (Debian/Ubuntu)
wget https://github.com/TheCorrectSynovian/SwordigoDesktop/releases/download/v2/swordigo-desktop_2.0.0_amd64.deb

# Install
sudo dpkg -i swordigo-desktop_2.0.0_amd64.deb

# Place your game assets in the install directory
mkdir -p assets/

# Run!
swordigo-desktop
⚠ Known issue: Background music may be less reliable in the AppImage build compared to building from source.

System dependencies required (usually pre-installed on most distros):

SDL2
Window management, input handling
OpenAL
3D audio, sound effects
OpenGL
Desktop renderer (Mesa / NVIDIA / AMD)
Unicorn Engine
ARMv7 emulation (bundled in AppImage)
# Ubuntu / Debian
sudo apt install libsdl2-2.0-0 libopenal1 libgl1

# Fedora / RHEL
sudo dnf install SDL2 openal-soft mesa-libGL

# Arch Linux
sudo pacman -S sdl2 openal mesa
⚠ IMPORTANT: This project does not distribute original game assets or binaries. You must provide your own from a legally-owned copy of Swordigo.

Game assets must be placed in the assets/ directory next to the executable:

swordigo-desktop            # or swordigo_boot if built from source
libswordigo.so              # v1.4.6 game binary (required)
libswordigo_test.so         # v1.4.12 game binary (optional)
assets/
  resources/

Extract assets from a legally-obtained copy of the Swordigo APK using a tool like apktool.

🔨 Build from Source

For the best experience (especially audio), build from source:

# Clone the repository
git clone https://github.com/TheCorrectSynovian/SwordigoDesktop.git
cd SwordigoDesktop

# Ensure you have the dependencies: unicorn, sdl2, openal, libgl
# Then build:
make swordigo_boot

# Run
./swordigo_boot

Known Issues

Bug Save Data

Save data is currently not written correctly and may not persist between sessions. Progress will be lost on exit.

Status: Active development — future release priority.

Bug Background Music

Background music playback is unreliable and may fail to start. More common in the AppImage build due to audio backend differences.

Workaround: Build from source for better audio stability.

Minor Graphical Artifacts

Minor graphical artifacts may occasionally appear. These are related to the GLES 1.x to desktop OpenGL translation layer.

Status: Being investigated and improved.

Partial Android-Specific Features

Some Android-specific features (achievements, cloud saves, in-app purchases) remain stubbed or partially implemented.

These are Android/Google Play Services features not relevant to desktop.
